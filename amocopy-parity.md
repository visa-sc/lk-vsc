# Паритетный обход копии amoCRM (crm.voyotravel.ru ↔ живой amo)

Методика (введена 21.07.2026 после справедливого замечания Андрея — расхождения вёрстки и
поведения находил он, а не оркестр): каждый раздел сверяется по ТРЁМ измерениям:
1. **Пиксели** — скриншот amo ↔ скриншот копии, дифф глазами (не метриками!)
2. **Поведение** — карта кликов: каждый элемент amo → что открывает; то же в копии
3. **Данные** — числа до рубля (сверка через amo API пофактно, тайлам amo не верить — их кэш врёт)

Инструменты: amo — вкладка Chrome (только чтение/клики по фильтрам; ≤1 rps на API, днём API
не дёргать после 429 21.07); копия — встроенный браузер; числа — /leads_spec, widget_stat2, БД.

- Порядок карточек в колонке: amo хранит ручной drag-порядок (в API недоступен) — копия сортирует
  по updated_at. Не воспроизводимо, принято.
